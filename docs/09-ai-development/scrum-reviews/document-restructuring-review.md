## 핵심 원칙 (3 Agent 공통 합의)

1. **"무엇을 만들었는가" ≠ "어떻게 만들었는가"** — 시스템 설계/기능 섹션에 개발 프로세스 정보를 혼재시키지 않는다
2. **사실 기반 서술만 허용** — Co-Authored 커밋 1건, Claude Code 생성 표기 3건이라는 실제 증거 범위 내에서 서술
3. **"도구로서의 AI" 관점 유지** — "AI가 모든 것을 했다"가 아닌 "개발자가 AI를 도구로 능숙하게 활용했다"는 주체성 유지
4. **기존 문서 톤/구조 일관성** — 각 문서의 번호 체계, 글머리 스타일, 격식체 수준을 그대로 따름
5. **정량 지표 변경 금지** — 측정된 성능 데이터, 완료된 작업 기록, 문서 변경 이력은 수정 불가
